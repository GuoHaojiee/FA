## 3. 工具链路说明

```
op_test_template/test_fa_builtin.cpp   （调用 aclnn API）
        │
        ▼ bisheng 编译，链接 libruntime_camodel.so（替代真实 runtime）
op_test_template/test_fa_builtin       （host 可执行程序）
        │
        ▼ msprof op simulator 包裹执行
        │
        ├─ CAModel 周期精确仿真 Ascend950PR_9599
        │       └─ CANN built-in FlashAttentionScore kernel
        │
        └─ 产出 → op_test_template/msprof_output/
                    ├── trace.json              （Perfetto timeline）
                    ├── visualize_data.bin      （可视化）
                    └── core*.{cube,vec}core*_instr_exe.csv  （指令级数据）
```

**关键点：**
- `libruntime_camodel.so` 替代真实 NPU runtime，使程序在 x86 主机上跑仿真。
- msprof 读取 CAModel 的周期计数，产出与真实硬件 msprof 格式相同的 CSV/JSON。
- 仿真 Wall time 远大于真实执行时间（CAModel 本身是软件模拟），`Total tick` 才是有效指标。

## 5. 分步手动操作

### 5.1 设置环境

```bash
source /home/guohaojie/Ascend/cann-9.0.0/set_env.sh
export ASCEND_HOME_PATH=/home/guohaojie/Ascend/cann-9.0.0
export LD_LIBRARY_PATH="/home/guohaojie/Ascend/cann-9.0.0/tools/simulator/Ascend950PR_9599/lib:${LD_LIBRARY_PATH}"
```

### 5.2 编译

```bash
cd /home/guohaojie/Guo/FA原生算子性能分析
rm -rf build && mkdir build && cd build

cmake -DRUN_MODE=sim -DSOC_VERSION=Ascend950PR_9599 ..
make -j$(nproc)
```

编译成功后二进制在：`op_test_template/test_fa_builtin`

### 5.3 运行性能仿真

```bash
cd /home/guohaojie/Guo/FA原生算子性能分析

OUTPUT_DIR=op_test_template/msprof_output
mkdir -p "$OUTPUT_DIR"

msprof op simulator \
    --application="$(pwd)/op_test_template/test_fa_builtin" \
    --soc-version=Ascend950PR_9599 \
    --output="$OUTPUT_DIR" \
    --aic-metrics=PipeUtilization \
    --timeout=300
```

> `--timeout=300`：单 kernel 仿真超时秒数，ViT 大 shape 可能需要调大（或内存不足）。  
> `--aic-metrics`：可选 `PipeUtilization`、`ArithmeticUtilization`、`Memory`，多个逗号分隔。

### 5.4 查看 Total tick

```bash
# 运行日志中直接搜索
grep "Total tick" /tmp/msprof_*.log 2>/dev/null || true

# 或从产物 CSV 统计最大 cycles 列
PROF_DIR=$(ls -td op_test_template/msprof_output/OPPROF_*/simulator 2>/dev/null | head -1)
awk -F',' 'NR>1 {sum+=$5} END {print "sum cycles:", sum}' \
    "${PROF_DIR}/core0.veccore0/core0.veccore0_instr_exe.csv"
```

## 6. 修改测试场景

测试源文件：`op_test_template/test_fa_builtin.cpp`

### 6.1 当前两个场景

| 场景 | B | S | H | headNum | headDim | scale |
|------|---|---|---|---------|---------|-------|
| 场景1 ViT | 160 | 576 | 1024 | 16 | 64 | 0.125 |
| 场景2 FTT | 16 | 30 | 192 | 8 | 24 | 0.2041 |

> 场景1 CAModel 内存占用约 6.5 GB，低内存机器建议降 B。

### 6.2 修改 shape 参数

在 `test_fa_builtin.cpp` 中找到对应函数 `TestScenario2_FTT` 或 `TestScenario1_ViT`，修改：

```cpp
int64_t B       = 2;       // Batch 大小（CAModel 仿真建议压小）
int64_t S       = 30;      // SeqLen
int64_t H       = 192;     // HiddenSize = headNum × headDim
int64_t headNum = 8;
double scaleValue = 0.2041; // 1/sqrt(headDim)，headDim=24 → 1/√24
```

修改后重新编译即可（无需改 CMakeLists.txt）：

```bash
cd build && make -j$(nproc)
```

### 6.3 只运行单一场景

在 `main()` 函数中注释掉不需要的场景：

```cpp
// TestScenario1_ViT(stream);   // 注释掉场景1
ret = TestScenario2_FTT(stream);
```

### 6.4 切换 RUN_MODE（真实 NPU）

> 需要连接真实 Ascend 硬件，当前环境为仿真模式。

```bash
cd build
cmake -DRUN_MODE=npu -DSOC_VERSION=Ascend950PR_9599 ..
make -j$(nproc)
# 直接运行，不用 msprof（真机用 msprof op --application=...）
```

## 7. 分析产物

### 7.1 用 analyze_trace.py 分析 trace

```bash
# 找到最新一次产物目录
PROF_DIR=$(ls -td op_test_template/msprof_output/OPPROF_*/simulator 2>/dev/null | head -1)

# 分析全局 trace
python3 analyze_trace.py "${PROF_DIR}/trace.json"

# 分析单核 trace（粒度更细）
python3 analyze_trace.py "${PROF_DIR}/core0.veccore0/trace.json"
```

脚本输出：
- 总执行 cycles 数
- 各功能单元（CUBE / VEC / MTE1 / MTE2 / MTE3 / FLOWCTRL）占比
- 耗时 Top-20 指令列表

### 7.2 手动查看指令级 CSV

```bash
PROF_DIR=$(ls -td op_test_template/msprof_output/OPPROF_*/simulator 2>/dev/null | head -1)

# 按 cycles 降序排列，查看最耗时的 10 条指令
sort -t',' -k5 -rn "${PROF_DIR}/core0.veccore0/core0.veccore0_instr_exe.csv" | head -11
```

CSV 列说明：

| 列名 | 含义 |
|------|------|
| `instr` | 指令名 |
| `addr` | 指令地址 |
| `pipe` | 流水线（CUBE/VEC/MTE1/MTE2/MTE3/SCALAR/FLOWCTRL） |
| `call_count` | 执行次数 |
| `cycles` | 总 cycles 消耗 |
| `running_time(us)` | 等效微秒（基于仿真频率） |
| `detail` | 附加参数信息 |

### 7.3 用 Perfetto 可视化 trace

```bash
# 将 trace.json 上传到 https://ui.perfetto.dev 即可查看 timeline
PROF_DIR=$(ls -td op_test_template/msprof_output/OPPROF_*/simulator 2>/dev/null | head -1)
echo "Trace 路径: ${PROF_DIR}/trace.json"
```

### 7.4 多核对比（找关键路径）

veccore 是当前瓶颈（比 cubecore 慢约 9%）。对比各核 cycles：

```bash
PROF_DIR=$(ls -td op_test_template/msprof_output/OPPROF_*/simulator 2>/dev/null | head -1)

for csv in "${PROF_DIR}"/core0.*/core0.*_instr_exe.csv; do
    total=$(awk -F',' 'NR>1 {sum+=$5} END {print sum}' "$csv")
    echo "$(basename $(dirname $csv))  → ${total} cycles"
done | sort -t'→' -k2 -rn | head -10
```

## 8. 常见问题排查

### Q1：`ASCEND_HOME_PATH not set` 编译报错

```bash
source /home/guohaojie/Ascend/cann-9.0.0/set_env.sh
export ASCEND_HOME_PATH=/home/guohaojie/Ascend/cann-9.0.0
```

### Q2：`libruntime_camodel.so: cannot open shared object file`

```bash
export LD_LIBRARY_PATH="/home/guohaojie/Ascend/cann-9.0.0/tools/simulator/Ascend950PR_9599/lib:${LD_LIBRARY_PATH}"
```

确认文件存在：

```bash
ls /home/guohaojie/Ascend/cann-9.0.0/tools/simulator/Ascend950PR_9599/lib/libruntime_camodel.so
```

### Q3：运行后没有出现 `[TmSim]` 日志

说明 CAModel 未生效，程序链接了真实 runtime 而非 `libruntime_camodel.so`。检查：

```bash
ldd op_test_template/test_fa_builtin | grep runtime
# 期望看到 libruntime_camodel.so，不应看到 libruntime.so
```

如果链接了错误的库，重新编译并确保 `RUN_MODE=sim`：

```bash
cd build && cmake -DRUN_MODE=sim .. && make -j$(nproc)
```

### Q4：msprof 超时 / 内存不足

- 减小 `B`（Batch）参数，场景2 B=2 约需 1 GB，B=16 约需 8 GB。
- 增大 `--timeout` 参数（默认 300 秒，ViT 大 shape 需要更长）。
- 只运行场景2（FTT），注释掉场景1。

### Q5：`opapi_transformer: library not found`

```bash
ls /home/guohaojie/Ascend/cann-9.0.0/x86_64-linux/lib64/libopapi_transformer.so
```

如果不存在，说明该 CANN 版本目录结构不同，在 lib64 下搜索：

```bash
find /home/guohaojie/Ascend/cann-9.0.0 -name "libopapi_transformer.so" 2>/dev/null
```

找到后修改 `CMakeLists.txt` 中的 `link_directories` 路径。

### Q6：产物目录在哪里

```bash
ls -td op_test_template/msprof_output/OPPROF_*/simulator | head -3
```

每次运行会生成带时间戳的新目录，旧目录不会自动删除。
